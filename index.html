Son el punto de partida de cualquier análisis de lingüística de corpus, ya que permiten conocer el tamaño real de los textos y normalizar las frecuencias antes de cualquier comparación.

1.2 Riqueza léxica (TTR y RTTR)

Nota

¿Qué es la riqueza léxica? El Type-Token Ratio (TTR) mide la proporción de palabras únicas respecto al total de palabras. Un TTR alto indica mayor variedad en el vocabulario. El Root TTR (RTTR) corrige el efecto del tamaño del texto, haciendo las comparaciones más justas entre discursos de distinta longitud.

4 Análisis de palabras clave (keyness)

La keyness identifica las palabras cuya frecuencia en el corpus foco (Sánchez) difiere significativamente del corpus de referencia (Rajoy). Para cada palabra se calcula: (1) el log-likelihood (LL), que mide si la diferencia es estadísticamente significativa; y (2) el log-ratio (LR), que mide cuántas veces más frecuente es en un corpus que en otro.

Solo se consideran palabras clave aquellas con LL ≥ 15,13 (p < 0,0001), que es el umbral estándar de Sketch Engine, y que aparezcan al menos en el 50% de los textos del corpus foco.

Ver código 
# ── Funciones reutilizables ──────────────────────────────────

# Log-likelihood (Rayson & Garside, 2000)
calc_LL <- function(O1, O2, N1, N2) {
  total <- O1 + O2
  E1 <- total * N1 / (N1 + N2)
  E2 <- total * N2 / (N1 + N2)
  LL <- 2 * (ifelse(O1 > 0, O1 * log(O1 / E1), 0) +
             ifelse(O2 > 0, O2 * log(O2 / E2), 0))
  round(LL, 2)
}

# Log-ratio (Hardie, 2014) — tamaño del efecto
calc_LR <- function(O1, O2, N1, N2) {
  round(log2(((O1 + 0.5) / N1) / ((O2 + 0.5) / N2)), 3)
}

# Simple Maths de Sketch Engine (add-N smoothing)
calc_SM <- function(O1, O2, N1, N2, add_N = 1000) {
  round(((O1 + add_N) / N1) / ((O2 + add_N) / N2), 4)
}

# Etiqueta de significación
etiqueta_sig <- function(LL) {
  case_when(
    LL >= 15.13 ~ "****  p<0.0001",
    LL >= 10.83 ~ "***   p<0.001",
    LL >= 6.63  ~ "**    p<0.01",
    LL >= 3.84  ~ "*     p<0.05",
    TRUE        ~ "n.s."
  )
}

# ── Cálculo manual de keyness ────────────────────────────────
# Agrupamos los textos en dos sub-corpora: Sánchez (foco) y Rajoy (referencia).
# Para cada tipo léxico calculamos LL, LR y SM.

# Frecuencias brutas por orador
dfm_grp <- dfm_group(dfm_corp, groups = docvars(dfm_corp, "orador"))
N_san   <- ntoken(dfm_grp["Sánchez", ])
N_raj   <- ntoken(dfm_grp["Rajoy",   ])

# Convertir a data frame largo
frecs <- convert(dfm_grp, to = "data.frame") |>
  pivot_longer(-doc_id, names_to = "palabra", values_to = "frec") |>
  pivot_wider(names_from = doc_id, values_from = frec, values_fill = 0) |>
  rename(f_san = Sánchez, f_raj = Rajoy) |>
  filter(f_san + f_raj >= 3)   # descartar hápax y cuasi-hápax

# Dispersión: % de documentos del corpus foco donde aparece cada palabra
docs_san <- docnames(dfm_corp)[docvars(dfm_corp, "orador") == "Sánchez"]
n_docs_san <- length(docs_san)

disp_san <- map_dbl(frecs$palabra, function(p) {
  if (!p %in% featnames(dfm_corp)) return(0)
  sum(dfm_corp[docs_san, p] > 0) / n_docs_san
})

# Tabla completa de keyness
keyness_tbl <- frecs |>
  mutate(
    dispersion_san = round(disp_san, 2),
    LL   = mapply(calc_LL, f_san, f_raj, N_san, N_raj),
    LR   = mapply(calc_LR, f_san, f_raj, N_san, N_raj),
    SM   = mapply(calc_SM, f_san, f_raj, N_san, N_raj),
    sig  = etiqueta_sig(LL),
    ptw_san = round(f_san / N_san * 1000, 3),
    ptw_raj = round(f_raj / N_raj * 1000, 3)
  ) |>
  arrange(desc(abs(LL)))

4.2 Mapa léxico: LL × log-ratio

Tip

Cómo leer este gráfico: cada punto es una palabra. El eje Y muestra la significación estadística (LL): cuanto más arriba, más fiable es la diferencia. El eje X muestra el tamaño del efecto (log-ratio): los puntos a la derecha son más frecuentes en Sánchez; los de la izquierda, en Rajoy. La línea discontinua marca el umbral LL = 15,13 (p < 0,0001).

Ver código 
# El gráfico LL × LR es la visualización estándar en CADS (corpus-assisted
# discourse analysis). Permite identificar de un solo vistazo qué palabras
# son al mismo tiempo significativas y de gran efecto.

kw_plot <- keyness_tbl |>
  filter(LL >= 6.63) |>    # mostrar palabras al menos ** significativas
  mutate(
    grupo = case_when(
      LR > 0 & LL >= 15.13 ~ "Clave Sánchez",
      LR < 0 & LL >= 15.13 ~ "Clave Rajoy",
      TRUE                  ~ "Tendencia (no umbral)"

6 Colocaciones

Las colocaciones son pares de palabras que co-ocurren con más frecuencia de la que cabría esperar por azar, dentro de una ventana de ±k palabras alrededor de un nodo. Se analizan aquí los colocados más fuertes de las palabras clave más discriminantes: cambio (Sánchez) y reforma (Rajoy). Se calculan tres medidas de asociación complementarias:

  • MI (Mutual Information): detecta colocaciones raras pero fijas, propias de la fraseología especializada.
  • t-score: detecta colocaciones frecuentes y gramaticalmente productivas.
  • logDice: equilibra ambos extremos y es más robusto con frecuencias bajas.

# Función genérica para calcular colocaciones de un nodo dado.
# Devuelve siempre un tibble con columna 'nodo', incluso si está vacío.
calcular_colocaciones <- function(nodo, ventana = 3, min_cooc = 3) {
  toks_sel <- tokens_select(toks_full, nodo,
                            selection = "keep",
                            window    = ventana,
                            padding   = TRUE)
  
  # textstat_collocations puede fallar si hay muy pocas co-ocurrencias
  col <- tryCatch(
    textstat_collocations(toks_sel, size = 2, min_count = min_cooc),
    error = function(e) NULL
  )
  
  if (is.null(col) || nrow(col) == 0) {
    return(tibble(collocation = character(), count = integer(),
                  lambda = numeric(), z = numeric(), nodo = nodo))
  }
  
  col |>
    as_tibble() |>
    filter(!str_detect(collocation,
                       paste0("^", nodo, " |", " ", nodo, "$"))) |>
    arrange(desc(lambda)) |>
    head(20) |>
    mutate(nodo = nodo)
}

7 Dispersión léxica

# Análisis de palabras clave (*keyness*) {#sec-keyness}

> La *keyness* identifica las palabras cuya frecuencia en el **corpus foco** (Sánchez) difiere significativamente del **corpus de referencia** (Rajoy). Para cada palabra se calcula: (1) el **log-likelihood** (LL), que mide si la diferencia es estadísticamente significativa; y (2) el **log-ratio** (LR), que mide cuántas veces más frecuente es en un corpus que en otro.
>
> Solo se consideran palabras clave aquellas con **LL ≥ 15,13** (*p* < 0,0001), que es el umbral estándar de Sketch Engine, y que aparezcan al menos en el **50% de los textos** del corpus foco.

```{r funciones-estadisticas}
# ── Funciones reutilizables ──────────────────────────────────

# Log-likelihood (Rayson & Garside, 2000)
calc_LL <- function(O1, O2, N1, N2) {
  total <- O1 + O2
  E1 <- total * N1 / (N1 + N2)
  E2 <- total * N2 / (N1 + N2)
  LL <- 2 * (ifelse(O1 > 0, O1 * log(O1 / E1), 0) +
             ifelse(O2 > 0, O2 * log(O2 / E2), 0))
  round(LL, 2)
}

# Log-ratio (Hardie, 2014) — tamaño del efecto
calc_LR <- function(O1, O2, N1, N2) {
  round(log2(((O1 + 0.5) / N1) / ((O2 + 0.5) / N2)), 3)
}

# Simple Maths de Sketch Engine (add-N smoothing)
calc_SM <- function(O1, O2, N1, N2, add_N = 1000) {
  round(((O1 + add_N) / N1) / ((O2 + add_N) / N2), 4)
}

# Etiqueta de significación
etiqueta_sig <- function(LL) {
  case_when(
    LL >= 15.13 ~ "****  p<0.0001",
    LL >= 10.83 ~ "***   p<0.001",
    LL >= 6.63  ~ "**    p<0.01",
    LL >= 3.84  ~ "*     p<0.05",
    TRUE        ~ "n.s."
  )
}
```


# Colocaciones {#sec-colocaciones}

> Las **colocaciones** son pares de palabras que co-ocurren con más frecuencia de la que cabría esperar por azar, dentro de una ventana de ±*k* palabras alrededor de un **nodo**. Se analizan aquí los colocados más fuertes de las palabras clave más discriminantes: *cambio* (Sánchez) y *reforma* (Rajoy). Se calculan tres medidas de asociación complementarias:
>
> - **MI** (*Mutual Information*): detecta colocaciones raras pero fijas, propias de la fraseología especializada.
> - **t-score**: detecta colocaciones frecuentes y gramaticalmente productivas.
> - **logDice**: equilibra ambos extremos y es más robusto con frecuencias bajas.

```{r colocaciones-funcion}
# Función genérica para calcular colocaciones de un nodo dado.
# Devuelve siempre un tibble con columna 'nodo', incluso si está vacío.
calcular_colocaciones <- function(nodo, ventana = 3, min_cooc = 3) {
  toks_sel <- tokens_select(toks_full, nodo,
                            selection = "keep",
                            window    = ventana,
                            padding   = TRUE)
  
  # textstat_collocations puede fallar si hay muy pocas co-ocurrencias
  col <- tryCatch(
    textstat_collocations(toks_sel, size = 2, min_count = min_cooc),
    error = function(e) NULL
  )
  
  if (is.null(col) || nrow(col) == 0) {
    return(tibble(collocation = character(), count = integer(),
                  lambda = numeric(), z = numeric(), nodo = nodo))
  }
  
  col |>
    as_tibble() |>
    filter(!str_detect(collocation,
                       paste0("^", nodo, " |", " ", nodo, "$"))) |>
    arrange(desc(lambda)) |>
    head(20) |>
    mutate(nodo = nodo)
}
```


# Dispersión léxica {#sec-dispersion}

> La **dispersión** mide cómo se distribuye una palabra a lo largo de los textos del corpus. Una palabra muy frecuente pero concentrada en un solo discurso no es representativa del estilo del orador. El **Deviation of Proportions** (DP) de Gries (2008) oscila entre 0 (perfectamente dispersa) y 1 (concentrada en un único texto).

```{r dispersion}
# Calculamos la dispersión de las palabras clave más relevantes
# usando la función textstat_frequency() de quanteda, que incluye

                                  groups = docvars(dfm_corp, "orador"))

  group_by(palabra, orador) |>
  summarise(
    n_docs      = n(),
    docs_presencia = sum(frec > 0),
    prop_docs   = round(docs_presencia / n_docs, 2),
    frec_media  = round(mean(frec), 2),
```

# Resumen metodológico {#sec-metodologia}

::: {.callout-note}
## Decisiones metodológicas

| Parámetro | Valor | Justificación |
|---|---|---|
| Umbral significación | LL ≥ 15,13 (*p* < 0,0001) | Estándar Sketch Engine; reduce falsos positivos en comparaciones múltiples |
| Umbral dispersión | ≥ 50% de textos del foco | Garantiza representatividad en todo el sub-corpus |
| Medida de efecto principal | Log-ratio (LR) | Interpretable en escala logarítmica; no depende del tamaño muestral |
| Medida de efecto alternativa | Simple Maths (add-N = 1000) | Más estable con palabras de baja frecuencia |
| Stopwords | Lista `stopwords("es")` de quanteda | Elimina palabras gramaticales para análisis de contenido |
| Ventana de colocación | ±4 palabras | Captura relaciones sintácticas directas |
| Mín. co-ocurrencias | 3 | Elimina colocaciones espurias o accidentales |
| N-gramas | Bigramas y trigramas | Sin stopwords; captura expresiones fijas de contenido |

:::
